# 01 · 项目概览与技术栈

## 这个项目是什么

**ShopMFE** 是一个用「微前端」架构实现的电商 Demo。它把一个完整的电商网站拆成 5 个独立的前端应用 + 1 个后端服务：

| 角色 | 名称 | 端口 | 职责 |
|------|------|------|------|
| 主应用（Host） | shell | 5173 | 整体外壳：顶部导航、底部、路由编排、购物车角标 |
| 微前端（Remote） | products | 5174 | 商品列表、商品详情 |
| 微前端（Remote） | cart | 5175 | 购物车 |
| 微前端（Remote） | auth | 5176 | 登录、注册 |
| 微前端（Remote） | order | 5177 | 结算下单、订单列表、订单详情 |
| 后端 | server | 3001 | 提供 REST API，数据存 SQLite |

用户访问 `http://localhost:5173`，看到的是一个完整网站。但背后，"商品"页的代码来自 5174，"购物车"页来自 5175……它们在浏览器里被**动态拼装**到一起。这就是微前端。

## 为什么要拆成微前端

想象一个大公司做电商：商品团队、购物车团队、订单团队各自独立。如果所有人改同一个代码仓库，会互相阻塞、发布耦合。微前端让每个团队：

- **独立开发**：各自一个仓库/应用，互不干扰
- **独立部署**：商品模块更新，只重启 5174，不影响别的
- **技术栈灵活**：理论上商品用 React、购物车用 Vue 都行（本项目统一用 React 降低学习成本）
- **增量上线**：新功能作为新微前端接入，老应用不用动

> 类比：微前端之于前端，就像微服务之于后端。

## 技术栈一览

### 前端

| 技术 | 作用 | 为什么选它 |
|------|------|-----------|
| **React 18** | UI 库 | 生态最大、就业面广 |
| **Vite 5** | 构建工具 + 开发服务器 | 比 Webpack 快很多，配置简单 |
| **@originjs/vite-plugin-federation** | 实现 Module Federation | 让 Vite 也能用 Webpack 5 的微前端能力 |
| **react-router-dom 6** | 前端路由 | 单页应用（SPA）必备，URL 切页面不刷新 |

### 后端

| 技术 | 作用 | 为什么选它 |
|------|------|-----------|
| **Node.js** | 运行 JS 的服务端环境 | 前后端同语言 |
| **Express** | Web 框架 | 最经典最小巧，适合学习 |
| **better-sqlite3** | 操作 SQLite 的 Node 库 | SQLite 是文件型数据库，零配置，同步 API 好理解 |
| **jsonwebtoken (JWT)** | 登录令牌签发与验证 | 无状态鉴权标准方案 |
| **cors** | 处理跨域 | 前端 5173 访问后端 3001 需要它 |

### 工程化

| 技术 | 作用 |
|------|------|
| **concurrently** | 一条命令同时启动 6 个服务 |
| **ESM**（import/export） | 现代 JS 模块语法，全项目使用 |

## 关键概念预告

读后面文档前，先有个印象（不懂没关系，07 会细讲）：

- **Module Federation（模块联邦）**：Webpack 5 提出的微前端方案。一个应用可以「暴露」自己的模块给别人用，也可以「引用」别人的模块。本项目的 `@originjs/vite-plugin-federation` 把这个能力搬到了 Vite。
- **Host / Remote**：主动加载别人的叫 Host（shell），被加载的叫 Remote（products 等）。
- **remoteEntry.js**：每个 Remote 编译后生成的"入口清单"文件，Host 通过它知道能加载哪些模块。
- **shared**：把 react 等公共依赖声明为"共享"，运行时只加载一份，避免重复。
- **SPA**：单页应用，所有页面在一个 HTML 里切换，靠 JS 改内容。
- **JWT**：登录后服务器发一个 token（一串字符），前端存起来，之后每次请求带上，服务器凭它认人。

## 项目目录结构

```
mfe/
├── package.json            # 根编排（concurrently 一键启动）
├── README.md               # 快速开始
├── docs/                   # 本文档
├── shared/                 # 5 个前端共享的代码（样式、API 封装、事件总线）
│   ├── styles.css          #   统一设计系统样式
│   ├── api.js              #   fetch 封装 + token 管理
│   └── bus.js              #   跨微前端事件总线
├── server/                 # 后端
│   ├── src/
│   │   ├── index.js        #   Express 入口
│   │   ├── db.js           #   SQLite 连接 + 建表
│   │   ├── seed.js         #   灌入演示数据
│   │   └── routes/         #   4 组路由：auth/products/cart/orders
│   └── shop.db             #   SQLite 数据库文件（运行后生成）
├── shell/                  # 主应用（Host）
│   └── src/{App,bootstrap,main,bus}.jsx + components/ + pages/
├── products/               # 商品微前端（Remote）
├── cart/                   # 购物车微前端（Remote）
├── auth/                   # 认证微前端（Remote）
└── order/                  # 订单微前端（Remote）
```

> 每个 `*/` 应用目录结构都一样：`package.json` + `vite.config.js` + `index.html` + `src/`。这是 Vite + React 的标准结构。
