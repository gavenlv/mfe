# 06 · 前端模块逐文件解析

把每个前端应用拆开讲：每个文件干什么、关键代码在哪。配合源码里的注释读。

> 建议读法：先读 `shared/`（公共基础）→ `shell/`（骨架）→ 4 个微前端（业务）。

---

## 一、shared/（共享模块）

被所有微前端通过相对路径引用，是项目的"公共地基"。

| 文件 | 作用 |
|------|------|
| `styles.css` | 统一设计系统：CSS 变量、按钮/卡片/表单/列表样式。所有微前端用同样的 class 名，外观就一致 |
| `api.js` | `api(path, options)` 统一 fetch 封装，自动带 JWT；`getToken/setAuth/getUser/clearAuth` 管理 localStorage 登录态 |
| `bus.js` | `window.__shopBus` 单例事件总线 + `EVENTS` 事件常量 |

> 为什么是相对路径而不是 npm 包？因为这是 Demo，简化依赖管理。生产可以发成 npm 包或用 monorepo。

---

## 二、shell/（主应用 / Host）

```
shell/src/
├── main.jsx              # 入口：import('./bootstrap')
├── bootstrap.jsx         # 挂载 React、引入全局样式
├── App.jsx               # 路由编排 + 动态加载 4 个微前端
├── bus.js                # 与 shared/bus.js 同源
├── components/
│   ├── Header.jsx        # 顶部导航 + 购物车角标 + 用户区
│   └── Footer.jsx        # 底部
└── pages/
    ├── Home.jsx          # 首页（shell 自己的页面）
    └── MultiInstance.jsx # 多实例演示页（同时装载多个子应用实例）
```

### `App.jsx` —— 全局大脑

核心职责：用 `BrowserRouter` 建立路由上下文，用 `Routes/Route` 把不同路径映射到不同组件，微前端用 `React.lazy` 动态加载。

```jsx
const ProductsApp  = lazy(() => import('productsApp/App'));   // 远程模块
const WorkspaceApp = lazy(() => import('workspaceApp/App')); // 嵌套联邦：workspace 又会加载 products/cart

<BrowserRouter>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/products/*" element={<Suspense><ProductsApp/></Suspense>} />
    <Route path="/cart"       element={...CartApp...} />
    <Route path="/auth/*"     element={...AuthApp...} />
    <Route path="/orders/*"   element={...OrderApp...} />
    <Route path="/checkout"   element={...OrderApp...} />
    <Route path="/workspace"  element={<Suspense><WorkspaceApp/></Suspense>} />  // 嵌套联邦
    <Route path="/multi"      element={<MultiInstance />} />                      // 多实例
  </Routes>
</BrowserRouter>
```

新增了 `WorkspaceApp`（`lazy` 引入 `workspaceApp/App`，由 workspace 微前端提供）和本地的 `MultiInstance` 页面。前者演示嵌套联邦，后者演示多实例。

亮点：
- `Suspense` 包 `lazy` 组件，加载期间显示"加载中…"
- 自定义 `ErrorBoundary`（类组件）捕获远程加载失败，显示友好提示
- `path="/*"` 通配符让子应用自己处理二级路由（如 /products/3）

### `components/Header.jsx` —— 全局状态汇聚点

Header 是通信机制的"集大成者"：

```jsx
useEffect(() => {
  // 1. 从 localStorage 恢复登录态（刷新后仍记得登录）
  const stored = localStorage.getItem('user');
  if (stored) setUser(JSON.parse(stored));

  // 2. 订阅事件总线
  const offCart   = bus.on(EVENTS.CART_CHANGED, setCartCount);
  const offLogin  = bus.on(EVENTS.LOGIN, u => { setUser(u); localStorage... });
  const offLogout = bus.on(EVENTS.LOGOUT, () => { setUser(null); ... });

  // 3. 拉取初始购物车数量
  fetchCartCount();

  // 4. 卸载时取消所有订阅（防内存泄漏）
  return () => { offCart(); offLogin(); offLogout(); };
}, []);
```

### `pages/Home.jsx`

纯展示首页，用 `Link` 跳转各模块，画了个 ASCII 架构图。

### `pages/MultiInstance.jsx` —— 多实例演示

核心思路：**用多个 Tab 同时装载多个子应用实例**，而不是只装一个再切 Tab。

- 3 个 products 实例（数码 / 电脑 / 穿戴）= 同一个子应用，靠 `initialCategory` 区分各自数据
- 1 个 cart 实例 = 不同子应用
- 关键：所有实例**同时挂载**，用 `display:none` 切换（不是条件渲染卸载），保留各自的内部状态（已选分类、搜索词、购物车内容等）

```jsx
const ProductsApp = lazy(() => import('productsApp/App'));
const CartApp     = lazy(() => import('cartApp/App'));

// 同一个 lazy 组件渲染多次 = 多个独立实例，各自的 props/state 互不影响
{tabs.map((t, i) => (
  <div key={t.key} style={{ display: active === i ? 'block' : 'none' }}>
    {t.type === 'products'
      ? <ProductsApp initialCategory={t.initialCategory} embedded />
      : <CartApp embedded />}
  </div>
))}
```

为什么用 `display:none` 而不是 `{active === i && <Comp/>}`？因为条件渲染会**卸载**不活跃的实例，状态丢失、远程模块重新加载。`display:none` 让 DOM 一直留着，状态也留着，切回来接着用。

> `embedded` 让子应用以"组件"形式嵌入：关闭导航跳转、隐藏标题，详见后文 products/cart 部分。

## 三、products/（商品微前端）

```
products/src/
├── main.jsx / bootstrap.jsx     # 同 shell，异步边界
├── App.jsx                       # 路径分发：/products → 列表，/products/:id → 详情
└── components/
    ├── ProductList.jsx           # 商品列表（搜索/分类/加购）
    └── ProductDetail.jsx         # 商品详情（数量/加购/立即购买）
```

### `App.jsx` —— 内部路由分发

不写 `BrowserRouter`，用 `useLocation` 读 Shell 当前路径，自己决定渲染哪个组件。新增 `embedded` / `initialCategory` 两个 props 支持"当组件嵌入"：

```jsx
export default function ProductsApp({ initialCategory, embedded } = {}) {
  const location = useLocation();  // 必须无条件调用
  if (embedded) return <ProductList initialCategory={initialCategory} embedded />;
  const match = location.pathname.match(/^\/products\/(\d+)$/);
  if (match) return <ProductDetail id={match[1]} />;
  return <ProductList />;
}
```

`embedded=true` 时直接渲染 `ProductList`，不做路由分发（嵌入场景下没必要也不该跳详情），并把 `initialCategory` 透传给列表。

> **`useLocation` 必须无条件调用**：这是 Hooks 规则——不能在 `if (embedded) return ...` 之前就 `return`，否则 `useLocation` 调用顺序在嵌入/非嵌入两种情况下不一致，React 会报错。所以即便 embedded 模式用不到 `location`，也要先调一次。

### `ProductList.jsx` 关键点

- `useEffect` 依赖 `[category, q]`：分类或搜索词变化才重新请求
- `URLSearchParams` 构造查询串：`?category=手机&q=air`
- 受控输入：`<input value={q} onChange={e=>setQ(e.target.value)} />`
- 卡片点击跳详情：`onClick={() => navigate('/products/' + p.id)}`
- 加购按钮 `e.stopPropagation()`：防止冒泡触发卡片跳转
- 加购后 `bus.emit(EVENTS.CART_CHANGED, count)` 通知角标

新增 `embedded` / `initialCategory` 两个 props，让列表能作为"组件"被嵌入：

```jsx
// 接受初始分类，默认 'all'
const [category, setCategory] = useState(initialCategory || 'all');

// embedded 时隐藏标题
{!embedded && <h1 className="page-title">全部商品</h1>}

// embedded 时禁用卡片点击跳详情（嵌入场景不该把用户带走）
onClick={embedded ? undefined : () => navigate(`/products/${p.id}`)}
```

### `ProductDetail.jsx` 关键点

- 数量增减：`setQty(q => Math.min(q+1, product.stock))`
- "立即购买" = 加购 + `navigate('/checkout')`

## 四、cart/（购物车微前端）

只有 `src/App.jsx` 一个组件（功能集中）。新增 `embedded` prop，支持作为"组件"被嵌入。

### 关键逻辑

```jsx
// 加载：没登录显示登录提示；否则 GET /api/cart
async function load() {
  if (!getToken()) return;       // 未登录直接返回，UI 层显示登录提示
  const data = await api('/cart');
  setData(data);
}

// 任何变更后都要：1) 重新拉购物车 2) 通知角标
async function refresh() {
  const data = await api('/cart');
  setData(data);
  const count = data.items.reduce((s,i)=>s+i.quantity, 0);
  bus.emit(EVENTS.CART_CHANGED, count);
}

// 改数量
async function updateQty(productId, qty) {
  await api('/cart/'+productId, { method:'PUT', body:{quantity:qty} });
  refresh();
}
```

### 条件渲染顺序

未登录 → 加载中 → 出错 → 空购物车 → 购物车列表。每种状态对应不同 UI。

### `embedded` 模式

`embedded=true` 时隐藏标题和底部的"清空 / 去结算"操作栏（"去结算"会 `navigate('/checkout')` 把用户切走路由，在嵌入场景下不该这么做），只保留列表 + 增删 + 合计：

```jsx
export default function CartApp({ embedded } = {}) {
  // ...已有的加载/增删逻辑不变

  // embedded 时隐藏标题
  {!embedded && <h1 className="page-title">购物车</h1>}

  // embedded 时隐藏底部操作栏（清空购物车 / 去结算）
  {!embedded && (
    /* 清空购物车 / 去结算 操作栏 */
  )}
}
```

## 五、auth/（认证微前端）

```
auth/src/
├── App.jsx                  # 路径分发：/auth/login → Login，/auth/register → Register
└── components/
    ├── Login.jsx
    └── Register.jsx
```

### Login.jsx 核心流程

```jsx
async function onSubmit(e) {
  e.preventDefault();                 // 阻止表单默认提交（刷新页面）
  try {
    const { token, user } = await api('/auth/login', {
      method:'POST', body:{ username, password }
    });
    setAuth(token, user);             // 存 localStorage
    bus.emit(EVENTS.LOGIN, user);     // 通知 Shell Header
    navigate('/');                    // 跳首页
  } catch (err) {
    setError(err.message);            // 显示错误（如"用户名或密码错误"）
  }
}
```

Register 几乎一样，多一个 name 字段，注册成功即登录。

## 六、order/（订单微前端）

```
order/src/
├── App.jsx                         # 路径分发：/checkout / /orders / /orders/:id
└── components/
    ├── Checkout.jsx                # 结算页
    ├── OrderList.jsx               # 订单列表
```

### App.jsx 路径分发

```jsx
const { pathname } = useLocation();
if (pathname === '/checkout') return <Checkout />;
const m = pathname.match(/^\/orders\/(\d+)$/);
if (m) return <OrderDetail id={m[1]} />;

```

### Checkout.jsx 核心

```jsx
async function onSubmit(e) {
  e.preventDefault();
  const { orderId } = await api('/orders', {
    method:'POST', body:{ address, phone }
  });
  bus.emit(EVENTS.CART_CHANGED, 0);        // 购物车已清空，通知角标归零
  navigate('/orders/' + orderId);          // 跳订单详情
}
```

### OrderList.jsx / OrderDetail.jsx

标准的"请求数据 → 渲染列表/详情"，带加载/错误/空态。订单状态用不同颜色标签（`.order-status.pending/.paid/.shipped`）。

## 七、workspace/（工作台微前端 · 嵌套联邦）

workspace 是本项目新增的**嵌套联邦**示范：它**既是 Remote（exposes `./App` 给 shell）又是 Host（remotes `productsApp/cartApp`）**，自己被 shell 加载的同时又内部加载 products/cart。

```
workspace/src/
├── main.jsx / bootstrap.jsx   # 同其它微前端，异步边界
└── App.jsx                     # 嵌套加载 products + cart
```

### `App.jsx` —— 嵌套加载的核心

左右两栏布局：左栏 `ProductsApp`（限定 `initialCategory="数码"`、`embedded`），右栏 `CartApp`（`embedded`）。两侧都不包 `BrowserRouter`，直接复用 shell 的路由上下文。

```jsx
import { lazy, Suspense } from 'react';

const ProductsApp = lazy(() => import('productsApp/App'));
const CartApp     = lazy(() => import('cartApp/App'));

export default function WorkspaceApp() {
  return (
    <div style={{ display: 'flex', gap: 16 }}>
      <div style={{ flex: 1 }}>
        <Suspense fallback={<div>加载中…</div>}>
          <ProductsApp initialCategory="数码" embedded />
        </Suspense>
      </div>
      <div style={{ flex: 1 }}>
        <Suspense fallback={<div>加载中…</div>}>
          <CartApp embedded />
        </Suspense>
      </div>
    </div>
  );
}
```

### 为什么 workspace 不包 BrowserRouter

workspace 自己也是被 shell 通过 `import('workspaceApp/App')` 加载的远程模块，运行在 shell 的 `BrowserRouter` 上下文里。如果 workspace 再包一层 `BrowserRouter`，就会出现两个 Router 实例，`useLocation/useNavigate` 拿到的上下文对不上，路由状态错乱。所以 workspace 只用 `useLocation/useNavigate` **接入** shell 路由，不自己建 Router。

> 整条加载链是：shell → workspace → products/cart，三层嵌套联邦。`shared` 让全局只有一份 React，所以 hooks、context、router 全程贯通。详见 [03 · 嵌套联邦](./03-Module-Federation原理详解.md#进阶嵌套联邦一个应用同时是-host-和-remote)。

## 通用模式总结

几乎每个微前端都遵循这套模式：

```
1. main.jsx → 动态 import bootstrap（异步边界）
2. bootstrap.jsx → createRoot + 引入 shared/styles.css
3. App.jsx → useLocation 读路径，分发到子组件
4. 组件 → useEffect 请求数据，useState 存状态，条件渲染
5. 跨应用通信 → bus.emit / bus.on + localStorage
6. 跳转 → useNavigate / Link
7. embedded 模式 → 子应用通过 embedded / initialCategory props 支持"当组件嵌入"，关闭导航跳转、隐藏标题/操作栏，可被嵌入到任意 Host（shell 页面 / workspace / 多实例页都可复用同一份子应用代码）
```

读懂一个微前端，其余都同构。
