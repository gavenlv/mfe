# 06 · 前端模块逐文件解析

把每个前端应用拆开讲：每个文件干什么、关键代码在哪。配合源码里的注释读。

> 建议读法：先读 `shared/`（公共基础）→ `shell/`（骨架）→ 4 个微前端（业务）。

---

## 一、shared/（共享模块）

被所有微前端通过相对路径引用，是项目的"公共地基"。

| 文件 | 作用 |
|------|------|
| `styles.css` | 统一设计系统：CSS 变量、按钮/卡片/表单/列表样式。所有微前端用同样的 class 名，外观就一致 |
| `api.js` | `api(path, options)` 统一 fetch 封装，自动带 JWT；`getToken/setAuth/getUser/clearAuth` 管理 localStorage 登录态 |
| `bus.js` | `window.__shopBus` 单例事件总线 + `EVENTS` 事件常量 |

> 为什么是相对路径而不是 npm 包？因为这是 Demo，简化依赖管理。生产可以发成 npm 包或用 monorepo。

## 二、shell/（主应用 / Host）

```
shell/src/
├── main.jsx              # 入口：import('./bootstrap')
├── bootstrap.jsx         # 挂载 React、引入全局样式
├── App.jsx               # 路由编排 + 动态加载 4 个微前端
├── bus.js                # 与 shared/bus.js 同源
├── components/
│   ├── Header.jsx        # 顶部导航 + 购物车角标 + 用户区
│   └── Footer.jsx        # 底部
└── pages/
    └── Home.jsx          # 首页（shell 自己的页面）
```

### `App.jsx` —— 全局大脑

核心职责：用 `BrowserRouter` 建立路由上下文，用 `Routes/Route` 把不同路径映射到不同组件，微前端用 `React.lazy` 动态加载。

```jsx
const ProductsApp = lazy(() => import('productsApp/App'));  // 远程模块

<BrowserRouter>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/products/*" element={<Suspense><ProductsApp/></Suspense>} />
    <Route path="/cart"       element={...CartApp...} />
    <Route path="/auth/*"     element={...AuthApp...} />
    <Route path="/orders/*"   element={...OrderApp...} />
    <Route path="/checkout"   element={...OrderApp...} />
  </Routes>
</BrowserRouter>
```

亮点：
- `Suspense` 包 `lazy` 组件，加载期间显示"加载中…"
- 自定义 `ErrorBoundary`（类组件）捕获远程加载失败，显示友好提示
- `path="/*"` 通配符让子应用自己处理二级路由（如 /products/3）

### `components/Header.jsx` —— 全局状态汇聚点

Header 是通信机制的"集大成者"：

```jsx
useEffect(() => {
  // 1. 从 localStorage 恢复登录态（刷新后仍记得登录）
  const stored = localStorage.getItem('user');
  if (stored) setUser(JSON.parse(stored));

  // 2. 订阅事件总线
  const offCart   = bus.on(EVENTS.CART_CHANGED, setCartCount);
  const offLogin  = bus.on(EVENTS.LOGIN, u => { setUser(u); localStorage... });
  const offLogout = bus.on(EVENTS.LOGOUT, () => { setUser(null); ... });

  // 3. 拉取初始购物车数量
  fetchCartCount();

  // 4. 卸载时取消所有订阅（防内存泄漏）
  return () => { offCart(); offLogin(); offLogout(); };
}, []);
```

### `pages/Home.jsx`

纯展示首页，用 `Link` 跳转各模块，画了个 ASCII 架构图。

## 三、products/（商品微前端）

```
products/src/
├── main.jsx / bootstrap.jsx     # 同 shell，异步边界
├── App.jsx                       # 路径分发：/products → 列表，/products/:id → 详情
└── components/
    ├── ProductList.jsx           # 商品列表（搜索/分类/加购）
    └── ProductDetail.jsx         # 商品详情（数量/加购/立即购买）
```

### `App.jsx` —— 内部路由分发

不写 `BrowserRouter`，用 `useLocation` 读 Shell 当前路径，自己决定渲染哪个组件：

```jsx
const { pathname } = useLocation();
const match = pathname.match(/^\/products\/(\d+)$/);
if (match) return <ProductDetail id={match[1]} />;
return <ProductList />;
```

### `ProductList.jsx` 关键点

- `useEffect` 依赖 `[category, q]`：分类或搜索词变化才重新请求
- `URLSearchParams` 构造查询串：`?category=手机&q=air`
- 受控输入：`<input value={q} onChange={e=>setQ(e.target.value)} />`
- 卡片点击跳详情：`onClick={() => navigate('/products/' + p.id)}`
- 加购按钮 `e.stopPropagation()`：防止冒泡触发卡片跳转
- 加购后 `bus.emit(EVENTS.CART_CHANGED, count)` 通知角标

### `ProductDetail.jsx` 关键点

- 数量增减：`setQty(q => Math.min(q+1, product.stock))`
- "立即购买" = 加购 + `navigate('/checkout')`

## 四、cart/（购物车微前端）

只有 `src/App.jsx` 一个组件（功能集中）。

### 关键逻辑

```jsx
// 加载：没登录显示登录提示；否则 GET /api/cart
async function load() {
  if (!getToken()) return;       // 未登录直接返回，UI 层显示登录提示
  const data = await api('/cart');
  setData(data);
}

// 任何变更后都要：1) 重新拉购物车 2) 通知角标
async function refresh() {
  const data = await api('/cart');
  setData(data);
  const count = data.items.reduce((s,i)=>s+i.quantity, 0);
  bus.emit(EVENTS.CART_CHANGED, count);
}

// 改数量
async function updateQty(productId, qty) {
  await api('/cart/'+productId, { method:'PUT', body:{quantity:qty} });
  refresh();
}
```

### 条件渲染顺序

未登录 → 加载中 → 出错 → 空购物车 → 购物车列表。每种状态对应不同 UI。

## 五、auth/（认证微前端）

```
auth/src/
├── App.jsx                  # 路径分发：/auth/login → Login，/auth/register → Register
└── components/
    ├── Login.jsx
    └── Register.jsx
```

### Login.jsx 核心流程

```jsx
async function onSubmit(e) {
  e.preventDefault();                 // 阻止表单默认提交（刷新页面）
  try {
    const { token, user } = await api('/auth/login', {
      method:'POST', body:{ username, password }
    });
    setAuth(token, user);             // 存 localStorage
    bus.emit(EVENTS.LOGIN, user);     // 通知 Shell Header
    navigate('/');                    // 跳首页
  } catch (err) {
    setError(err.message);            // 显示错误（如"用户名或密码错误"）
  }
}
```

Register 几乎一样，多一个 name 字段，注册成功即登录。

## 六、order/（订单微前端）

```
order/src/
├── App.jsx                         # 路径分发：/checkout / /orders / /orders/:id
└── components/
    ├── Checkout.jsx                # 结算页
    ├── OrderList.jsx               # 订单列表
    └── OrderDetail.jsx             # 订单详情
```

### App.jsx 路径分发

```jsx
const { pathname } = useLocation();
if (pathname === '/checkout') return <Checkout />;
const m = pathname.match(/^\/orders\/(\d+)$/);
if (m) return <OrderDetail id={m[1]} />;
return <OrderList />;
```

### Checkout.jsx 核心

```jsx
async function onSubmit(e) {
  e.preventDefault();
  const { orderId } = await api('/orders', {
    method:'POST', body:{ address, phone }
  });
  bus.emit(EVENTS.CART_CHANGED, 0);        // 购物车已清空，通知角标归零
  navigate('/orders/' + orderId);          // 跳订单详情
}
```

### OrderList.jsx / OrderDetail.jsx

标准的"请求数据 → 渲染列表/详情"，带加载/错误/空态。订单状态用不同颜色标签（`.order-status.pending/.paid/.shipped`）。

## 通用模式总结

几乎每个微前端都遵循这套模式：

```
1. main.jsx → 动态 import bootstrap（异步边界）
2. bootstrap.jsx → createRoot + 引入 shared/styles.css
3. App.jsx → useLocation 读路径，分发到子组件
4. 组件 → useEffect 请求数据，useState 存状态，条件渲染
5. 跨应用通信 → bus.emit / bus.on + localStorage
6. 跳转 → useNavigate / Link
```

读懂一个微前端，其余都同构。
