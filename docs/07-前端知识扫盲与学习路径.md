# 07 · 前端知识扫盲与学习路径

这一章是"字典 + 路线图"。读前面文档遇到不懂的名词，来这里查。最后给一条推荐学习顺序。

## 一、名词速查

### 工程基础

| 名词 | 解释 |
|------|------|
| **Node.js** | 让 JS 能在浏览器外（服务器）运行的环境。本项目后端和构建工具都靠它 |
| **npm** | Node 的包管理器，`npm install` 装依赖，`npm run xxx` 跑脚本 |
| **package.json** | 项目的"身份证"：名字、依赖、脚本。`dependencies` 是运行依赖，`devDependencies` 是开发期依赖 |
| **node_modules** | 装下来的依赖都堆在这，别手动改，别提交到 git |
| **ESM** | ES Modules，`import/export` 语法。比老的 `require` 更现代。本项目 `"type":"module"` 启用 |
| **Vite** | 构建工具+开发服务器。dev 模式秒开（不打包，按需编译），build 模式用 Rollup 打包 |
| **vite.config.js** | Vite 的配置文件，配插件、端口、构建选项 |

### React

| 名词 | 解释 |
|------|------|
| **组件** | 返回 UI（JSX）的函数。`function App() { return <div>...</div> }` |
| **JSX** | JS 里写 HTML 的语法糖。`<div className="x">{count}</div>` |
| **Hook** | React 提供的函数，让函数组件拥有状态等能力。都以 `use` 开头 |
| **useState** | 声明状态。`const [count, setCount] = useState(0)`，改状态用 setCount，组件自动重渲染 |
| **useEffect** | 副作用钩子。请求数据、订阅事件、操作 DOM 放这里。第二个参数是依赖数组，变化时重新执行；`[]` 表示只执行一次 |
| **依赖数组** | useEffect/useMemo 的第二个参数。里面列的变量变了，回调才重跑。空数组=只在挂载时跑一次 |
| **lazy + Suspense** | `lazy(() => import())` 懒加载组件，`<Suspense fallback>` 包裹显示加载态 |
| **useNavigate** | react-router 的钩子，`navigate('/path')` 编程式跳转 |
| **useLocation** | 读当前 URL 信息（pathname、search 等） |
| **Link** | 渲染成 `<a>` 但不刷新页面的路由链接：`<Link to="/cart">` |
| **useParams** | 读动态路由参数（如 `/products/:id` 里的 id） |
| **受控组件** | input 的 value 绑定 state，onChange 更新 state。单向数据流 |
| **条件渲染** | `{loading ? <Loading/> : <Content/>}`，根据状态显示不同 UI |
| **key** | 列表渲染时每个元素的唯一标识，帮 React 高效更新 DOM |

### 路由

| 名词 | 解释 |
|------|------|
| **SPA** | 单页应用。整个网站只有一个 HTML，切页面靠 JS 改内容，不刷新 |
| **BrowserRouter** | 用 HTML5 history API 实现的路由，URL 长得像正常路径（/products/3） |
| **Routes / Route** | 路由表。`<Route path="/cart" element={<Cart/>} />`：访问 /cart 渲染 Cart |
| **通配符 `*`** | 匹配任意。`/products/*` 匹配 /products/、/products/3 等 |

### 网络

| 名词 | 解释 |
|------|------|
| **HTTP 方法** | GET 查、POST 增、PUT 改、DELETE 删 |
| **HTTP 状态码** | 2xx 成功、4xx 客户端错（401 未登录/404 不存在/400 参数错）、5xx 服务器错 |
| **REST / RESTful** | 一种 API 设计风格：URL 表资源、方法表操作 |
| **fetch** | 浏览器原生发 HTTP 请求的函数，返回 Promise |
| **Promise / async-await** | 处理异步。`async` 函数里 `await` 一个 Promise，写起来像同步 |
| **请求头 / 响应头** | HTTP 消息的元信息。如 `Authorization: Bearer xxx`、`Content-Type: application/json` |
| **CORS** | 跨域资源共享。浏览器默认拦跨域请求，后端加特定响应头才放行 |
| **JSON** | JS 对象表示法，前后端交换数据的事实标准 |

### 鉴权

| 名词 | 解释 |
|------|------|
| **JWT** | JSON Web Token，登录令牌。服务器签发，前端保存，请求带上，服务器验签认人 |
| **Authorization 头** | 标准的携带凭证请求头，格式 `Bearer <token>` |
| **localStorage** | 浏览器本地存储，同源共享，刷新不丢。本项目存 token 和 user |
| **无状态鉴权** | 服务器不存 session，每次凭 token 验证。适合多实例、前后端分离 |

### 微前端 / 构建

| 名词 | 解释 |
|------|------|
| **微前端** | 把一个大前端拆成多个独立应用，运行时拼装。类比微服务 |
| **Module Federation** | 模块联邦，Webpack 5 的微前端方案。本项目用其 Vite 移植版 |
| **Host / Remote** | 加载方 / 被加载方 |
| **remoteEntry.js** | Remote 编译后生成的入口清单文件 |
| **exposes / remotes / shared** | 暴露 / 引用 / 共享依赖，MF 三大配置 |
| **异步边界** | `import()` 制造的异步点，让 shared 依赖先初始化再执行业务代码 |
| **chunk** | 打包产物里的一块代码文件，按需加载 |

### 后端 / 数据库

| 名词 | 解释 |
|------|------|
| **Express** | Node.js Web 框架 |
| **中间件** | 请求处理链条上的一环，能改 req/res、决定是否 next() |
| **SQLite** | 文件型数据库，零配置，一个 .db 文件就是整个库 |
| **better-sqlite3** | Node 操作 SQLite 的库，同步 API |
| **SQL 注入** | 把用户输入拼进 SQL 导致的安全漏洞。用占位符 `?`/`@x` 防御 |
| **事务 transaction** | 一组操作要么全成功要么全回滚，保证数据一致 |
| **外键 FK** | 表间关联，如 cart_items.user_id 指向 users.id |
| **主键 PK** | 表里每行的唯一标识 |

### CSS

| 名词 | 解释 |
|------|------|
| **CSS 变量** | `--primary: #0071e3`，用 `var(--primary)` 引用，方便统一改主题 |
| **flex** | 一维弹性布局，`display:flex` |
| **grid** | 二维网格布局，`display:grid` |
| **响应式** | `@media (max-width: 720px)` 根据屏幕宽度应用不同样式 |
| **盒模型** | 每个元素是 content+padding+border+margin 的盒子 |

## 二、推荐学习路径

按这个顺序看本项目，循序渐进：

### 第 1 步：跑起来，有个感性认识
1. 读根目录 `README.md`，按"快速开始"装依赖、build、start
2. 打开 http://localhost:5173，点点看：浏览商品 → 登录（demo/demo123）→ 加购 → 结算 → 看订单
3. 读 `docs/01`、`docs/02`，建立整体认知

### 第 2 步：补前端基础（按需）
- 完全不懂 React → 先看 `docs/07` 的 React 部分，再去看 `shell/src/App.jsx` 和 `Home.jsx`
- 不懂路由 → 看 `docs/05` 的"共享依赖"部分 + react-router 的用法
- 不懂 fetch/async → 看 `shared/api.js` 注释

### 第 3 步：读后端
1. `server/src/db.js` —— 看表结构，理解数据模型
2. `server/src/seed.js` —— 看怎么灌数据
3. `server/src/index.js` —— 看 Express 中间件、JWT 验证
4. `server/src/routes/products.js` —— 最简单的 CRUD
5. `server/src/routes/auth.js` —— 登录签发 token
6. `server/src/routes/cart.js` —— 联表查询
7. `server/src/routes/orders.js` —— 事务（最难，最重要）
8. 配合 `docs/04`

### 第 4 步：读共享层
1. `shared/api.js` —— fetch 封装 + token 管理
2. `shared/bus.js` —— 事件总线
3. `shared/styles.css` —— 设计系统
4. 配合 `docs/05`

### 第 5 步：读主应用 shell
1. `shell/vite.config.js` —— Module Federation 的 remotes
2. `shell/src/main.jsx` + `bootstrap.jsx` —— 异步边界
3. `shell/src/App.jsx` —— 路由编排、lazy、Suspense、ErrorBoundary
4. `shell/src/components/Header.jsx` —— 全局状态汇总
5. 配合 `docs/03`、`docs/06`

### 第 6 步：读 4 个微前端
1. products（最完整，列表+详情）
2. cart（增删改查 + 事件通知）
3. auth（登录态流程）
4. order（下单事务的前端侧）
任选一个细读，其余同构。配合 `docs/06`。

### 第 7 步：进阶思考
- 如果要新增一个"优惠券"微前端，要改哪些地方？（答：新建一个 app + 暴露 ./App + shell 加路由 + remotes + 必要的 shared）
- 如果要把后端换成 MySQL，要改哪些？（答：换驱动，SQL 基本通用）
- 微前端的缺点是什么？（答：构建复杂、运行时开销、调试难、样式隔离要自己处理）

## 三、动手练习建议

1. **加一个商品分类页**：在 products 微前端加 `/products/category/:cat` 路由
2. **加商品库存显示**：详情页显示库存，库存为 0 时禁用加购
3. **加订单状态流转**：后端加"发货"接口，订单列表显示状态
4. **加一个新微前端**：比如"评价"模块，走一遍完整接入流程
5. **把密码哈希**：后端用 `bcrypt` 替换明文存储

每改完跑 `npm run build && npm start` 验证。
